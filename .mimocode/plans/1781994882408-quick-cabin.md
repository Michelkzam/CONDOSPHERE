# Plano de Testes e Correções - CondoSphere Reestruturado

## Problemas Identificados

### CRÍTICOS (sistema não funciona sem correção)

1. **Login com bug de lógica** (`index.html:248`)
   - A expressão `pass === user.split('.').pop() === 'geral' ? '12345678901' : pass` tem precedência incorreta
   - Qualquer usuário/senha é aceito (o else-if sempre passa para `showApp()`)
   - **Fix**: Validar senha corretamente com CPF do preset

2. **Toasts das páginas filhos não aparecem no pai**
   - Páginas usam `parent.postMessage({type:'toast',msg:...}, '*')` mas o index.html não escuta `message` events
   - **Fix**: Adicionar event listener no index.html para mensagens do iframe

3. **Referência `dbClient` inexistente** (`index.html:336`)
   - `window.parentAPI.getDB` retorna `dbClient` que não existe no shell
   - **Fix**: Remover ou condicionar a referência

4. **CSS não carrega nas páginas filhos via Vercel**
   - Caminho `/src/styles/condosphere.css` funciona no server.js mas pode falhar em produção Vercel
   - **Fix**: Testar e ajustar caminhos relativos se necessário

### MÉDIOS (funcionalidade degradada)

5. **JS original perdido** - Todas as funções CRUD, Supabase, xlsx, CNAB, etc. estavam no monolito e não foram migradas para as páginas
   - As páginas são apenas templates visuais estáticos
   - **Fix**: Migrar JavaScript do backup para as páginas correspondentes (grande esforço)

6. **Sidebar toggle não colapsa corretamente**
   - O botão `&#9664;` não se adapta quando sidebar está collapsed
   - **Fix**: Melhorar ícone do toggle

7. **Vercel retornou 401** - Projeto pode ter proteção de deploy habilitada
   - **Fix**: Verificar configurações do projeto Vercel

### BAIXOS (cosmético)

8. **Ícone sidebar** - Usa `&#9664;` (triângulo) ao invés de hamburger
9. **Sem responsividade** - Não há menu mobile (hamburger button)
10. **Páginas com body padding variável** - Algumas usam 24px, outras 0px
